## Numbering/naming convention

- Epic: Ex - E.g. E1 or E17 - number of the epic. Always prefixed with capital E and referred to using E and the number, e.g. E1 - and the Epic name if relevant
- Feature: Fe-x lives inside en epic file listed above. Always Prefixed with capital F and epic number - and a running number. E.g. F1-1 feature 1 under epic 1.
- Bug: Be-x-y - Also lives inside Epic files, always prefixed with a capital B: an epic number - feature number - running number. E.g. B2-2-1 - Bug number 1, inside feature 2 of Epic 2. 

## Status legend
- **Done** — shipped and validated
- **Partially done** — some features complete, see epic file for detail
- **In progress** — active development
- **Planned** — scoped but not started
- **Future** — explicitly deferred past MVP
